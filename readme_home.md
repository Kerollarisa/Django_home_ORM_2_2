# Выполнение задания "M2M relations"  

1. **Модели (models.py)**  

    Создана модель Section для тематических разделов  

    Создана промежуточная модель ArticleSection со связью Many-to-Many между статьями и разделами  

    Добавлено поле is_main для отметки основного раздела  

    В модель Article добавлен метод get_sections_ordered(), который возвращает разделы в порядке: сначала основной, затем остальные по алфавиту  

    2. **Админ (admin.py)**  

    Зарегистрирована модель Section для управления разделами  

    В админку статей добавлен ArticleSectionInline для редактирования связей  

    Создан ArticleSectionInlineFormset с переопределенным методом clean()  

    Реализована валидация: проверка на наличие одного и только одного основного раздела (ошибка при 0 или >1)  

    3. **Шаблон (news.html)**  

    В карточку каждой статьи добавлен вывод тегов  

    Используется метод get_sections_ordered() для правильной сортировки  

    Основной раздел выделен стилем badge-primary, остальные - badge-secondary  

    4. **Результат**  

    В админ при редактировании статьи можно добавлять разделы и указывать основной  

    Валидация не позволяет сохранить статью без основного раздела или с несколькими основными  

    На главной странице под каждой новостью отображаются теги разделов в правильном порядке: сначала основной, затем остальные по алфавиту
